# TrendMap User Guide

TrendMap helps a strategy or innovation team turn weak signals into reviewed trends, monitor those trends over time, and convert them into assumptions, scenarios, options, and decision-ready recommendations.

The product is organised into four workflow areas:

- **Phase 1: Discover** - set the market context, review sources, extract signals, approve trends, and read insight summaries.
- **Phase 2: Monitor** - configure monitoring, run scans, and review alerts.
- **Phase 3: Strategize** - translate approved trends into assumptions, scenarios, strategic options, decision briefs, and a roadmap.
- **Phase 4: Intelligence Hub** - run and observe the agent team cycle, debates, and predictions.

## Basic Navigation

1. Use the left sidebar to choose a workflow area: **Discover**, **Monitor**, **Strategize**, or **Intelligence Hub**.
2. Use the tab bar across the top of the main content area to move between screens inside the selected workflow.
3. Work from left to right within each workflow. Later screens depend on earlier screens having usable data.

## Recommended End-to-End Workflow

### 1. Set Up The Industry Context

Open **Phase 1: Discover > Industry Setup**.

Use this screen to define the market you want TrendMap to analyse.

1. Enter the industry name.
2. Enter the geography.
3. Add a short description.
4. Add strategic priorities, customer segments, competitors, and time horizons.
5. Select **Save Configuration**.

Use this when you want the later analysis to stay anchored to a specific business context.

### 2. Review Sources

Open **Phase 1: Discover > Sources**.

Sources are the information feeds TrendMap uses to understand the market.

1. Review each source name, type, credibility, relevance, freshness, and status.
2. Select **Approve** for sources you trust.
3. Select **Reject** for sources that should not influence trend evidence.
4. Add analyst notes when a source needs context, such as bias, scope, or quality concerns.
5. Move to **Documents** when the source list is ready.

Approved sources feed the rest of the discovery workflow.

### 3. Extract Signals From Documents

Open **Phase 1: Discover > Documents**.

Documents are the raw items collected from sources.

1. Review the available documents.
2. Use the source visibility toggle if you need to inspect documents from all sources.
3. Select **Extract Signals** on documents that should be analysed.
4. Confirm that extracted documents show as completed.
5. Move to **Signals**.

Signals are observations or evidence fragments extracted from documents.

### 4. Review Signals And Generate Candidate Trends

Open **Phase 1: Discover > Signals**.

Use this screen to inspect extracted signals before clustering them into trends.

1. Filter by PESTLE category if needed.
2. Filter by minimum confidence if you only want stronger signals.
3. Open a signal to inspect its source and document trace.
4. Select **Generate Candidate Trends**.
5. Move to **Trends**.

Candidate trends are grouped patterns built from one or more signals.

### 5. Approve Or Reject Trends

Open **Phase 1: Discover > Trends**.

This is the main human review step before TrendMap uses trends for strategy.

1. Review candidate trend cards.
2. Open **Details** to inspect evidence, drivers, blockers, recommended actions, and score history.
3. Edit the trend name if needed.
4. Select **Approve** for trends that are credible and strategically useful.
5. Select **Reject** for weak, duplicate, or unsupported trends.
6. Move to **Insights** after the most important trends are approved.

Only approved trends should feed insight summaries, monitoring, and strategy analysis.

### 6. Read The Strategic Insight Brief

Open **Phase 1: Discover > Insights**.

Use this as the executive summary of approved trends.

1. Review the executive overview.
2. Read what changed, why it matters, and recommended actions.
3. Check opportunities, emerging risks, and watch items.
4. Use the brief to decide which trends deserve monitoring or deeper strategy work.

## Monitoring Workflow

### 7. Configure Monitoring Rules

Open **Phase 2: Monitor > Rules Engine**.

Monitoring rules define which approved sources TrendMap should keep watching.

1. Review approved sources available for monitoring.
2. Select **Enable Monitoring** for sources that should be tracked.
3. Choose the monitoring frequency.
4. Add keywords to focus scans on relevant market activity.
5. Disable monitoring for sources that should no longer be watched.

### 8. Run Monitoring And Review Scan Summaries

Open **Phase 2: Monitor > Dashboard**.

Use the dashboard to run monitoring and inspect system activity.

1. Review system health, last run, next run, sources monitored, documents scanned, and alerts generated.
2. Choose a simulation scenario if available.
3. Select **Run Monitoring Now** for an active rule.
4. Review the latest scan summary.
5. Follow any recommended actions.

### 9. Review Alerts

Open **Phase 2: Monitor > Alerts**.

Alerts identify changes that may need attention.

1. Filter alerts by severity.
2. Review alert title, summary, type, related trend, signals, documents, and sources.
3. Select **Acknowledge** after an alert has been reviewed.
4. Use important alerts to update assumptions, revisit trends, or trigger strategic action.

## Strategy Workflow

### 10. Review Or Edit Strategy Context

Open **Phase 3: Strategize > Strategy Context**.

The dashboard gives a top-level view of opportunities, threats, assumptions, recommended options, and indicators.

1. Start on **Dashboard** to review the current strategic summary.
2. Switch to **Context Settings** to edit the company profile, business model, risk appetite, goals, capabilities, constraints, and planning horizons.
3. Select **Save Context** after changes.

The strategy context shapes all downstream strategic analysis.

### 11. Generate And Review Assumptions

Open **Phase 3: Strategize > Assumptions**.

Assumptions describe what must be true for approved trends to matter strategically.

1. Select **Generate Assumptions**.
2. Review the generated assumptions.
3. Filter by assumption type if needed.
4. Update assumption status as evidence improves.

### 12. Monitor Leading Indicators

Open **Phase 3: Strategize > Indicators**.

Leading indicators help track whether assumptions are becoming more or less likely.

1. Review indicators linked to assumptions.
2. Check the monitoring question for each indicator.
3. Update indicator status as new information arrives.
4. Use accelerating indicators as early warnings for strategic decisions.

### 13. Generate Opportunities And Threats

Open **Phase 3: Strategize > Implications**.

Implications translate trends into strategic opportunities, threats, risks, and watch items.

1. Select **Generate Implications**.
2. Review opportunity, threat, risk, and watch counts.
3. Filter the view by implication type.
4. Prioritise items with high urgency, impact, and confidence.

### 14. Generate Scenarios

Open **Phase 3: Strategize > Scenarios**.

Scenarios describe possible futures based on implications and assumptions.

1. Select **Generate Scenarios**.
2. Review each scenario narrative.
3. Compare likelihood, impact, and time horizon.
4. Use scenarios to pressure-test strategic options.

### 15. Generate Strategic Options

Open **Phase 3: Strategize > Options**.

Strategic options are possible actions the organisation could take.

1. Select **Generate Options**.
2. Review options by type, such as invest, experiment, partner, monitor, defend, exit, or build capability.
3. Compare impact, feasibility, urgency, confidence, and time to value.
4. Accept or reject options as appropriate.

### 16. Generate A Decision Brief

Open **Phase 3: Strategize > Decision Brief**.

The decision brief is the leadership-ready summary.

1. Make sure implications, options, assumptions, and indicators have been generated.
2. Select **Generate Brief**.
3. Review the headline, executive summary, top opportunities, top threats, recommended options, assumptions to test, and indicators to monitor.
4. Use this brief to support a leadership discussion or decision meeting.

### 17. Build The Strategic Roadmap

Open **Phase 3: Strategize > Roadmap**.

The roadmap places accepted strategic options into planning horizons.

1. Make sure strategic options exist.
2. Select **Generate Roadmap**.
3. Review **Now**, **Next**, and **Later** roadmap items.
4. Update item status to proposed, accepted, in progress, or rejected.
5. Use the roadmap to track which strategic actions are moving forward.

## Intelligence Hub Workflow

### 18. Trigger The Agent Team Cycle

Open **Phase 4: Intelligence Hub > Agent Activity**.

This screen shows the autonomous intelligence team activity log.

1. Select **Trigger Global Cycle**.
2. Watch the live event log as agents run.
3. Wait for the cycle to complete.
4. Use the resulting activity to inspect debates and predictions.

### 19. Review Agent Debates

Open **Phase 4: Intelligence Hub > Debate Console**.

Debates show where agents disagree, challenge evidence, or reach consensus.

1. Review active debates.
2. Read each agent message.
3. Pay attention to validation concerns and confidence changes.
4. Use the consensus summary to decide whether a prediction or trend needs more evidence.

### 20. Review Prediction Timelines

Open **Phase 4: Intelligence Hub > Prediction Timeline**.

Predictions show forecasted trends across future horizons.

1. Review each predicted trend.
2. Compare likelihood and confidence.
3. Inspect drivers, dependencies, and critical assumptions.
4. Feed important predictions back into assumptions, monitoring, or strategic options.

## Traceability Health Check

Open **Traceability** from the sidebar.

Use this screen to verify that evidence chains are complete.

1. Select **Run Check**.
2. Review any issues grouped by type.
3. Fix missing links by returning to the relevant source, document, signal, or trend workflow.
4. Re-run the check after fixes.

A healthy chain should connect:

```text
Trend -> Signal -> Document -> Source
```

## Suggested Weekly Operating Rhythm

1. Review new source and document intake.
2. Extract new signals.
3. Generate or refresh candidate trends.
4. Approve only evidence-backed trends.
5. Run monitoring.
6. Review alerts and changed trend scores.
7. Update assumptions and indicators.
8. Refresh implications, scenarios, options, and decision brief.
9. Update the roadmap.
10. Run the agent cycle and inspect debates or predictions.

## Common Empty States

- **No documents** - approve sources first, or load more source content.
- **No signals** - extract signals from documents.
- **No candidate trends** - make sure signals exist and have approved evidence.
- **No insights** - approve trends first.
- **No monitoring rules** - enable monitoring for approved sources.
- **No assumptions or implications** - approve trends, then run the generators.
- **No decision brief** - generate upstream assumptions, implications, options, and indicators first.
- **No predictions** - run the global agent cycle from Agent Activity.

## Best Practices

- Keep rejected sources out of the evidence chain unless they are useful for bias checks.
- Add analyst notes whenever a source, trend, or assumption needs human context.
- Approve trends only when the evidence is clear enough to support strategic work.
- Treat low-confidence watch items as monitoring candidates, not decisions.
- Use the decision brief for leadership conversations and the roadmap for execution tracking.
- Run traceability checks before relying on outputs for important decisions.
